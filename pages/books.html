我的阅读书单

整理进度:50%

Target: 保证每周阅读量, 阅读, 不断阅读, 总会改变一些东西的 :)

工作三年, 统计了下一共买了一百五十多本, 目前看完三分之二的样子, 再买剁手, 且看且珍惜.....

大部分为与阅读或正在阅读的, 后续书单表示在阅读列表中但尚未阅读.

会标注好书/渣书, 供借鉴.

另, 技术方面我关注的是偏后端

技术

1. 架构和设计

大型网站技术架构:核心原理与案例分析 [DONE, 五星经典]

对整体系统如何设计及部署有一个框架性了解, 方向正确. 很概要, 具体细节需要自己去深究.

构建高性能的Web站点 [DONE, 五星经典]

结合上面那本书看, 针对性能, 详细地分析了各个环节的技术细节, 涉及很多底层的东西, 收获很大.

简约之美:软件设计之道 [DONE, 五星经典]

如其名, 这本书很薄很简约, 一百页左右, 针对复杂性做了一系列分析, 引出一些软件设计时需要注意的Rules

2. 基础

算法(第4版) [10%, 五星经典]

一直挣扎着想看, 一直拖延(太厚了好不好,囧)......近期加入列表, 看是阅读

图解HTTP [DONE*2, 四星半]

没有勇气去刷HTTP权威指南的童鞋, 可以看看这本书, 日本作者一个好处就是, 很多图, 很清晰. 对入门来说帮助较大. 当然, 刷高级的还得啃厚砖头.

图解TCP/IP [DONE, 四星半]

同上, 没有勇气刷TCP/IP详解的童鞋, 可以看这本

后续书单:

HTTP权威指南
TCP/IP 详解 卷1/2
大话数据结构
大话设计模式

3. 后端: Python

Python 基础教程 [DONE, 五星经典]

作为入门书籍来说, 很赞

Python 学习手册 [DONE*2, 四星半]

可以做入门, 或者查漏补缺, 或者手册用, 一个字: 全(厚).

Python 核心编程 [DONE, 三星]

买了, 看了, 完了, 送人了. 话说很多人拿着本作入门, 私以为一般...

像计算机科学家一样思考 Python [DONE, 四星]

对初学者来说还行, 稍微掌握了python的来看没啥...

Python 开发实战 [DONE, 三星]

日本一家公司出的书, 一些实际使用, 看看就行, 参考作用, 对初学者用处不大.

Django Web开发指南 [DONE, 三星]

很老的一本书了, Django都出了不知多少个新版本了. 我学Python后学习的第一个框架, 建议直接去看文档或者手册.

Python 源码剖析 [60%, 五星经典]

必读经典, 啃啃源码, 对后期来说收获很多. 在读, 画图.

Python 标准库 [手册, 四星半]

手册加实例, 偶尔翻翻用

Python Cookbook [手册, 四星]

手册, 偶尔翻

后续书单:

Python 网络编程
Python 高级编程

4. 后端: Golang

Go Web编程 [DONE, 四星半]

用来入门很合适, 开源的书籍, 不过还是买了纸质版的支持下

Go语言程序设计 [DONE, 四星半]

手册性质的, 查阅用......话说Golang的边边角角很容易忘了

Go语言编程 [DONE, 一星渣]

买的第一本, 那时候国内还没有.....实在不适合初学者, 不建议入, 有种告诉你1+1=2然后让你求偏微分的感觉.

5. 后端: linux

Linux Shell脚本攻略 [DONE, 五星经典]

Shell 脚本学习指南 [DONE, 四星半]

鸟哥的Linux私房菜 [DONE, 五星经典]

Vim使用技巧 [DONE, 五星]

好不容易看到一本关于 Vim 的书, 好评, 建议反复看和实验.

学习 Nginx HTTP Server [DONE, 四星]
